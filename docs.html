基本の使い方Basic Flow

  1. Unity上部メニューから Tools / N4NSY / AudioLink Reactive を開きます。
  2. Open Tools / N4NSY / AudioLink Reactive from the Unity top menu.
  3. Avatar欄に対象アバターを指定します。
  4. Assign the target avatar in the Avatar field.
  5. Avatar指定時に必要な項目が自動セットアップされます。
  6. Required items are auto-setup when avatar is assigned.
  7. レンダラー一覧を開きます。
  8. Open the renderer list.
  9. Avatar配下のRenderer付きGameObjectを選びます。
  10. Pick a renderer GameObject under the avatar.
  11. 選択したRendererをALR対象に追加します（重複追加なし）。
  12. Add the selected renderer as ALR target (no duplicate add).
  13. 対象Materialごとに設定します。
  14. Configure each target material.
  15. Previewで見た目を確認します。
  16. Check visuals with Preview.
  17. 必要に応じて設定を書き出し / 読み込みします。
  18. Export/import settings when needed.
  19. 必要に応じてアップロード反映ON/OFFを切り替えます。
  20. Toggle build/upload apply ON/OFF when needed.

Avatar

  • Avatar欄にはアバターRootを指定します。
  • Assign the avatar root in Avatar field.
  • 指定時に必要なALR Controllerが自動セットアップされます。
  • Required ALR Controller is auto-setup on assignment.
  • Toolsメニューから開いた場合は空の状態で開始します。
  • When opened from Tools menu, Avatar starts empty.
  • ALRコンポーネント側から開いた場合は該当Avatarが入った状態で開きます。
  • When opened from ALR component, the related avatar is pre-filled.
  • 対象Rendererがない状態で閉じると、条件により空Controllerが削除されます。
  • If closed with no target renderer, an empty controller may be removed depending on conditions.

Renderer List

  • Avatar配下のMeshRenderer / SkinnedMeshRenderer付きGameObjectを一覧表示します。
  • Lists MeshRenderer / SkinnedMeshRenderer GameObjects under avatar.
  • 表示名はRendererコンポーネントが付いているGameObject名を使います。
  • Display name uses the renderer GameObject name.
  • 一覧から選択したRendererをALR対象に追加できます。
  • You can add selected renderer to ALR targets.
  • 追加済みRendererは重複追加されません。
  • Already-added renderers are not duplicated.
  • 一覧名クリックでHierarchy上の該当GameObjectを選択/Pingします。
  • Clicking a list name selects/pings the matching GameObject in Hierarchy.
  • HierarchyでRenderer付きGameObjectを選ぶと、一覧側でも強調表示されます。
  • Selecting renderer objects in Hierarchy also highlights the matching list entry.

Target Material Cards

  • 対象Materialはカード形式で表示されます。
  • Target materials are shown as cards.
  • 見出しは Renderer名 / Material名 形式です。
  • Header format is Renderer / Material.
  • _ALReactive_Slot0 のような内部識別サフィックスは表示しません。
  • Internal suffixes like _ALReactive_Slot0 are hidden.
  • カード内の「編集されるマテリアル」「対象Renderer」の重複表示は省略されます。
  • Redundant in-card target labels are omitted.

反応の仕方と設定項目Reactions and Settings

光らせるGlow

音に合わせて発光を変化させます。反応音域 / 発光色 / 色・マスク / 不透明度から触ると分かりやすいです。

Changes emission with sound. Start with band, emission color, color/mask, and opacity.

  • 反応音域 / 発光色 / 色・マスク / 不透明度 / 点滅の強さ / グラデーション / 視差の強さ / 蛍光 / 詳細設定
膨らませるInflate

音に合わせてメッシュを膨らませます。初心者は反応音域とふくらみの強さから始めるのがおすすめです。

Inflates mesh with sound. Beginners should start from band and inflate strength.

  • 反応音域 / ふくらみの強さ / Correction / サイズ自動補正 / 詳細設定

Preview

  • プレビュー有効 / プレビュー量 を使って、Editor上で見た目を確認します。
  • Use Preview Enabled and Preview Amount to check visuals in the Editor.
  • プレビュー量で反応の強さを確認できます。
  • Preview Amount shows effect strength.
  • Play Mode復帰後に表示が崩れた場合は、更新ボタンで再同期します。
  • If display breaks after Play Mode return, resync with Refresh.

設定書き出し / 読み込み

  • Material単位で設定を書き出し / 読み込みできます。
  • Export/import is available per material.
  • ボタンは各MaterialカードではなくWindow側にあります。
  • Buttons are placed on window side, not inside each material card.
  • 実行後に対象Materialを選びます。
  • After pressing, select the target material.
  • 保存対象は設定値のみです（Avatar/Renderer/Scene Object/Material参照は保存しません）。
  • Only values are saved (no avatar/renderer/scene/material references).
  • 別アバターや別Materialへ設定値だけ流用できます。
  • You can reuse values on other avatars/materials.

アップロード反映ON/OFF / Runtime Toggle

  • アップロード反映ON/OFFは、ALR設定を保持したまま Build / Upload へ反映するか切り替える機能です。
  • Build/upload apply ON/OFF toggles whether ALR settings are applied while keeping settings data.
  • OFF時は設定は残りますがアップロードに反映しません。一時的に外したい時に使えます。
  • When OFF, settings remain but are not applied to upload.
  • Runtime Toggle / MA Menu Installer はVRChat内メニューでALR効果ON/OFFする機能で、現在は開発中・調整中です。
  • Runtime Toggle / MA Menu Installer is for in-game menu ON/OFF control, currently in development/tuning.

トラブルシュートTroubleshooting

  • Windowに設定UIが出ない場合は更新ボタンを押してください。
  • If UI does not appear, press Refresh.
  • Play Modeから戻ったあと表示が消えた場合も更新します。
  • If display disappears after Play Mode, refresh again.
  • Renderer一覧に出ない場合はAvatar欄を確認します。
  • If renderer list is empty, verify Avatar field.
  • 対象Materialがない場合は先にRendererを追加してください。
  • If no target material, add renderer first.
  • Rendererが重複追加されないのは仕様です。
  • No duplicate renderer add is expected behavior.
  • NDMF / Modular Avatar / lilToon / VRChat SDK のバージョン確認をしてください。
  • Check NDMF / Modular Avatar / lilToon / VRChat SDK versions.
  • Consoleエラーがある場合は問い合わせ時に添えてください。
  • Include full console errors when reporting issues.
  • Noto Sans CJK JP のログはNDMF側フォント読み込みログで、ALR本体の致命的エラーとは限りません。
  • Noto Sans CJK JP logs often come from NDMF font loading and are not always fatal ALR errors.
